# Speakers Feature Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add per-event speakers (with translated description and action_label) returned nested inside the `GET /event/:slug` response.

**Architecture:** Two new tables (`event_speakers`, `event_speakers_translations`) follow the exact pattern of `tickets` / `tickets_translations`. A new `EventSpeakerSerializer` is added and wired into `EventSerializer` as an `attribute :speakers` block.

**Tech Stack:** Rails 8.1, Alba serializers, RSpec + FactoryBot, PostgreSQL

```ruby
# app/serializers/event_speaker_serializer.rb

class EventSpeakerSerializer < ApplicationSerializer
  attributes :name, :action_url

  attribute :image do |object|
    ApplicationSerializer.asset_url(object.image)
  end

  attribute :description do |object|
    object.translations(params[:languages_code])&.description
  end

  attribute :action_label do |object|
    object.translations(params[:languages_code])&.action_label
  end
end
```
